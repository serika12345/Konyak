## Highlights

- Added a built-in Steam profile for macOS. Konyak can now install Steam from
  Profile Manager, prepare its bottle, bind the installed program to the
  profile, and launch it through the normal managed-program path.
- Added a data-driven built-in profile catalog and Profile Manager workflow.
  Automatic installation downloads checksum-pinned resources, applies declared
  dependencies in order, runs the installer, verifies the managed executable,
  and persists the profile binding only after installation succeeds. Manual
  profile application remains available for programs that are already
  installed.
- Added the macOS Steam launch rules needed for `steamwebhelper.exe`. A real
  Steam installation was exercised through Konyak's public app/CLI path and its
  login interface rendered instead of remaining a black window.
- The Steam profile installs `corefonts`, `vcrun2022`, native x86 and x64
  `d3dcompiler_47.dll` files, and `fakejapanese` before running the Steam
  installer. Native DLL downloads are SHA-256 and PE-machine verified, placed
  atomically, and do not create a Wine DLL override.

## Profile and Launch Improvements

- Profile installation now handles installers that start a long-running
  application, including Steam's **Run Steam** completion path, without leaving
  Konyak waiting on descendant output pipes.
- Bound profile rules are preserved when a program is launched from Profile
  Manager, from a normal pin, or from a pinned Windows shortcut. Executables
  stored as either Wine Windows paths or bottle-local host paths are mapped to
  the same managed program identity.
- Restored extracted Windows icons for programs pinned with `C:\...` paths
  while keeping their portable Windows-path identity in bottle metadata.
- Added safe recovery for bottles whose persisted profile metadata is invalid
  or incompatible. Konyak now keeps those bottles visible and offers an
  explicit repair flow instead of silently dropping them from the sidebar.
- Updated the managed macOS runtime line to
  `crossover-26.1.0-konyak.4` and aligned macOS development runtime selection
  with the latest published runtime source metadata.

future availability.

## Known Limitations

- The built-in Steam profile in v1.1.0 is supported on macOS only. Linux does
  not yet propagate profile child-process argument rules through its Wine
  runtime, so this release does not claim Steam profile support on Linux.
- Steam's profile-specific child-process rule avoids the observed black login
  window by disabling GPU use in `steamwebhelper.exe`. The underlying generic
  CEF/Chromium D3DMetal GPU-process failure is not yet fixed for other
  applications.

## Verification

- An existing manually installed Steam client was bound through Profile Manager

  and window evidence confirmed the compatibility rule was applied and the
  login UI rendered.
- Separately, the hosted macOS Profile Installation CLI Smoke passed against a
  published runtime using a repository-owned synthetic Windows fixture. It
  exercised checksum rejection, ordered HTTPS resources and pre-install
  actions, native DLL placement, automatic and manual profile application,
  idempotent reinstall, ordinary pinned and shortcut launches, child-process
  rules, metadata retention, and process termination.
- Post-merge `Konyak Verify` and the macOS profile-install smoke passed on the
  v1.1.0 source revision before release preparation.
