## Highlights

- Added user-managed compatibility profiles. Profile Manager can now import,
  edit, duplicate, export, and delete JSON profiles without requiring changes
  to Konyak itself.
- Profile edits are validated against Konyak's schema and semantic rules before
  they can be saved. Invalid fields and unsupported behavior are reported in
  the editor, and only validated manifests are written to the profile library.

## Profile Management

- Bundled profiles such as Steam remain read-only. They can be duplicated into
  an editable user profile when customization is needed.
- Profile updates and deletions detect changes made after editing began, which
  prevents an older editor session from silently overwriting newer profile
  content.
- Applying a profile now snapshots its launch-completion policy and child
  process compatibility rules into the bottle. Editing or deleting the source
  profile later does not unexpectedly change launches for an already configured
  bottle.
- Added versioned JSON CLI contracts for validating, importing, updating,
  exporting, inspecting, listing, and deleting user profiles.
- Invalid user profile files are reported individually without hiding valid
  bundled profiles from the catalog.

## Documentation

- Published the compatibility profile authoring guide, validation rules,
  versioning policy, and generated Schema v1 reference in the
  [compatibility profile documentation](https://serika12345.github.io/Konyak/profiles/).

## Compatibility and Known Limitations

- Existing Schema v1 profiles and older bottle metadata remain compatible; this
  release does not introduce a profile-schema breaking change.
- `childProcessRules` and `installerCompletion` are currently executed only by
  the Konyak macOS Wine runtime. Profiles containing those fields can be
  represented and managed on Linux, but those behaviors are not yet applied by
  the Linux runtime.
- macOS release artifacts are ad-hoc signed and are not notarized.

## Verification

- Profile management is covered by CLI contract, profile-library I/O, Flutter
  client, widget, and golden tests, including invalid-editor and delete-feedback
  states on macOS and Linux.
- The release candidate is verified through Konyak's full repository gates and
  the maintained packaged macOS release smokes before the release tag is
  published.
