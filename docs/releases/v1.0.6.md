## Highlights

- Completed the nullable and primitive boundary cleanup series across the
  Flutter app and CLI backend, replacing internal compatibility sentinels with
  explicit state, result, and selection models while keeping public CLI JSON
  stable.
- Hardened CLI/domain type safety with value objects and typed catalogs for
  runner kinds, runtime definitions, runtime source manifests, runtime install
  requests, and macOS version capability handling.
- Split large Flutter home-loader and CLI contract surfaces into smaller
  focused modules and tests, with governance checks that prevent the completed
  refactors from regressing.
- Added user-facing program log file settings and additional Wine logging
  presets for one-shot program runs and pinned program configuration.

## Verification

- Local release preparation runs `just release-candidate-gates` before creating
  the release commit and tag. On macOS this includes `just verify`, the macOS
  release build, packaged runtime extraction, DMG layout, Finder integration
  with PuTTY, packaged app CLI bridge, and app update handoff smokes.
- The GitHub Actions publish workflow verifies repository checks and release
  artifacts before publishing the macOS DMG and Linux AppImage assets.
