## Highlights

- Fixed program startup for applications that load assets relative to their
  executable. Regular programs, pinned programs, and resolved Windows shortcuts
  now use the target executable's parent directory as their default current
  working directory.
- Added a per-program custom working-directory setting. Custom paths are
  validated as bottle-portable `C:\` paths, resolved inside the bottle, and
  checked for existence before launch.

## Program Launch Improvements

- Applied one working-directory contract across macOS and Linux launch paths,
  including pinned launcher manifests and shortcuts whose target executable is
  resolved separately from the `.lnk` file.
- Added explicit machine-readable errors for invalid, missing, or unresolvable
  working directories. Konyak rejects an unresolvable path before invoking the
  runner or performing pre-launch side effects.
- Added maintained macOS and Linux public-runtime CLI smoke coverage for the
  executable-parent default, bottle-local custom `C:\` override, pinned
  launch, and resolved-shortcut launch paths.

## Fixes and Maintenance

- Fixed pinned-program selection feedback accumulating across multiple tiles.
  Selecting another pinned program now clears the previous tile's selection
  state while preserving double-click launch, bounce animation, and context
  menu behavior.

## Known Limitations

- macOS release artifacts are ad-hoc signed and are not notarized.
- The original Touhou executable from issue #62 was not rerun after the fix.
  A repository-owned Windows probe dynamically verified the same relative-data
  behavior through Konyak's public CLI and Wine runtime path.
