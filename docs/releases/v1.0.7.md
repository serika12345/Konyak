## Highlights

- Added GPTK/D3DMetal import support for user-provided Apple GPTK 4.x payloads
  alongside the existing GPTK 3.x import path. Konyak still does not bundle or
  redistribute Apple GPTK/D3DMetal binaries.
- Added explicit GPTK import version handling in Settings and the CLI. Users
  can choose Auto, GPTK 3, or GPTK 4, and Konyak reports the currently installed
  GPTK/D3DMetal major version after import.
- Added visible Settings feedback when a manual GPTK 3/4 selection does not
  match the selected payload, so an incorrect payload is rejected with a clear
  version mismatch message.
- Made macOS D3D10 fallback behavior CrossOver-compatible. D3D10 remains
  outside the GPTK/D3DMetal override path, DXVK remains the preferred D3D10
  route, and the base WineD3D/winevulkan route is preserved as the fallback
  when GPTK/D3DMetal is selected for a D3D10-only program.
- Preserved imported GPTK/D3DMetal components across macOS runtime
  reinstall/repair operations while keeping user-provided payloads isolated
  under the optional `components/gptk-d3dmetal` runtime component.

## Verification

  `crossover-26.1.0-konyak.0`.
- GPTK3/GPTK4 import compatibility was proved through the maintained public
  macOS CLI smoke, which installs fresh runtimes, imports Apple GPTK 3.0 and
  Apple GPTK 4.0 beta 1 payloads, verifies `list-runtimes --json`, and runs the
  maintained D3D10, D3D11, and D3D12 backend smoke targets.
- Runtime CI now includes the CrossOver-compatible WineD3D/winevulkan D3D10
  fallback smoke after assembled runtime artifact download, so the D3D10
  fallback path is verified without rebuilding the Wine runtime during smoke
  reruns.

  the release commit and tag.

  artifacts before publishing the macOS DMG and Linux AppImage assets.
