## Highlights

- Fixed a packaged app update issue where the macOS app could show the same
  Konyak update prompt again after successfully updating to v1.0.4.
- Synchronized the embedded CLI app version with the Flutter app version during
  release builds, so `check-app-update --json` compares the installed app
  against the latest release using the correct current version.
- Hardened release preparation and governance checks so future releases update
  both `pubspec.yaml` and the CLI app update version together.

## Verification

- Reproduced the v1.0.4 issue against `/Applications/Konyak.app`: the app bundle
  reported version `1.0.4`, but the packaged CLI reported
  `currentVersion=1.0.3` and `status=available` for `v1.0.4`.
- Verified the fixed CLI path by compiling with
  `-D KONYAK_APP_VERSION=1.0.4` and confirming `check-app-update --json`
  returns `currentVersion=1.0.4` and `status=current` against local v1.0.4
  release metadata.
- `just verify` passed before this release.
